## Documentation Standards

### File Naming Conventions

- **UPPERCASE.md** - Major documentation files
- **lowercase.md** - Supporting documentation
- **kebab-case.md** - Multi-word descriptive files

### Document Structure

All major documents should include:
1. Title and description
2. Table of contents (for long docs)
3. Last updated date
4. Quick links/navigation
5. Clear section headers
6. Code examples where applicable
7. Related documents section

## Contributing to Documentation

### Adding New Documentation

1. Place file in appropriate subdirectory
2. Update this INDEX.md with new entry

1. Update "Last Updated" date
